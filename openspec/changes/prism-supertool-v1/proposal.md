# Proposal: prism-supertool-v1

## Why
Prism v3 is solid but externally dependent. Its identity is spread across hand-maintained skill files, external CLI tools, and undocumented state contracts. To become a durable product-engineering system, Prism needs to own its core models, runtime, and skill generation — without losing what already works.

## What Changes
A phased transformation from externally-dependent skill to owned product-engineering system:
- Phase A: Unify internal truth (architecture doc, cleanup, documentation)
- Phase B: Upgrade product memory (owned document model)
- Phase C: Native supervisor (owned runtime)
- Phase D: Native reviews and verification
- Phase E: Skill/adapter compiler
- Phase F: Safe self-improvement engine

## Capabilities
- prism-supertool-v1: The governing change for the full transformation
